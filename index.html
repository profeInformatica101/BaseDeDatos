Base de datos relacional

Conceptos Básicos:

En una base de datos relacional, una tabla es una colección de registros estructurados en filas y columnas que representan una entidad o relación específica, como por ejemplo clientes, productos o ventas. Cada columna de una tabla representa un atributo o campo de la entidad, mientras que cada fila representa un registro único con valores para cada campo. Los campos de una tabla tienen un tipo de dato específico que determina el tipo de información que se puede almacenar en ellos.

Tabla
Es una colección de datos referentes al mismo tema o asunto. Por ejemplo, la base de datos de una tienda online podría tener las siguientes tablas (Productos, Pedidos, Clientes, Envíos...)
Registro
Se refieren a los items presentes en de cada tabla. Es decir, cada registro se corresponde a cada una de las filas de dicha tabla.
Campos
Aparece como una columna y contiene una categoría de información.

Ejemplo:

Tabla: Vehículos

Campo 1: MATRÍCULA Campo 2: MODELO Campo 3: KM
Registro 1 2345 HWK SEAT 23.001
Registro 2 3939 MNR TOYOTA 24.000
Registro 3 9090 PPH RENAULT 12.103

Tipos de datos

Los tipos de datos en una base de datos relacional son utilizados para definir el tipo de información que se almacenará en cada columna de una tabla. Cada sistema de gestión de bases de datos (DBMS) tiene su propia lista de tipos de datos disponibles, pero en general, los tipos de datos más comunes en las bases de datos relacionales incluyen:

  1. Enteros: Los tipos de datos enteros se utilizan para almacenar valores numéricos enteros, como el número de un ID o la edad de una persona. Algunos ejemplos de tipos de datos enteros incluyen INTEGER, SMALLINT, BIGINT, TINYINT, etc.
  2. Decimales: Los tipos de datos decimales se utilizan para almacenar valores numéricos con decimales, como las cantidades de dinero o las coordenadas geográficas. Algunos ejemplos de tipos de datos decimales incluyen FLOAT, DOUBLE, DECIMAL, NUMERIC, etc.
  3. Cadenas de caracteres: Los tipos de datos de cadenas de caracteres se utilizan para almacenar texto, como nombres, direcciones o descripciones. Algunos ejemplos de tipos de datos de cadenas de caracteres incluyen CHAR, VARCHAR, TEXT, etc.
  4. Fechas y horas: Los tipos de datos de fechas y horas se utilizan para almacenar valores de fecha y hora, como fechas de nacimiento o tiempos de registro. Algunos ejemplos de tipos de datos de fechas y horas incluyen DATE, TIME, DATETIME, TIMESTAMP, etc.
  5. Booleanos: Los tipos de datos booleanos se utilizan para almacenar valores verdadero/falso o sí/no. Normalmente solo ocupan un bit de memoria.
  6. Binarios: Los tipos de datos binarios se utilizan para almacenar datos en formato binario, como imágenes, sonidos o archivos. Algunos ejemplos de tipos de datos binarios incluyen BLOB, VARBINARY, IMAGE, etc.
  7. Otros: Además de los tipos de datos mencionados anteriormente, también existen otros tipos de datos que se utilizan para almacenar valores específicos, como direcciones de correo electrónico o direcciones IP. Algunos ejemplos de estos tipos de datos son VARCHAR(max), NVARCHAR(max), XML, JSON, etc.

Tipo Tamaño Definición Ejemplo
INTEGER 4 bytes Un número entero que puede ser positivo, negativo o cero. 23
FLOAT 4 bytes Un número decimal de precisión simple. 3.14
DOUBLE 8 bytes Un número decimal de precisión doble. 3.14159265359
VARCHAR(n) n bytes Una cadena de caracteres de longitud variable con un máximo de n caracteres. 'Hola'
DATE 3 bytes Una fecha en formato 'AAAA-MM-DD'. '2022-01-31'
TIME 3 bytes Una hora en formato 'HH:MM:SS'. '14:30:00'
DATETIME 8 bytes Una combinación de fecha y hora en formato 'AAAA-MM-DD HH:MM:SS'. '2022-01-31 14:30:00'
BOOLEAN 1 byte Un valor que puede ser verdadero o falso. TRUE

En general, es importante elegir el tipo de datos correcto para cada columna de una tabla, ya que esto puede afectar el rendimiento y la eficiencia de la base de datos. Además, una buena elección de tipos de datos puede ayudar a asegurar la integridad de los datos y evitar errores o pérdida de información.
